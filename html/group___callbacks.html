◆ m_onLiveDataAvailable

void(* XsDotCallbackPlainC::m_onLiveDataAvailable) (struct XsDotCallbackPlainC *, struct XsDotDevice *device, const struct XsDataPacket *packet)

Called when new data has been received from a device or read from a file.

This callback is for the Live stream, so there may be gaps in the data, but it will always contain the latest data.

Parameters
deviceThe device that initiated the callback.
packetThe data packet that has been received (and processed). This may be 0 when the callback originates from a non-device, such as the XsDataBundler.

◆ m_onProgressUpdated

void(* XsDotCallbackPlainC::m_onProgressUpdated) (struct XsDotCallbackPlainC *, struct XsDotDevice *device, int current, int total, const struct XsString *identifier)

Called when a long-duration operation has made some progress or has completed. Examples include device firmware upgrades. When current == total the operation has completed.

Parameters
deviceThe device that initiated the callback.
currentThe current progress.
totalThe total work to be done. When current equals total, the task is completed.
identifierAn identifier for the task. This may for example be a filename for file read operations.
